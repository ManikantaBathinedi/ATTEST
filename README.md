# ATTEST — Agent Testing & Trust Evaluation Suite

End-to-end testing framework for AI agents. Test any agent (Azure Foundry, HTTP APIs, callable functions) with assertions, LLM evaluators, and multi-turn conversations.

## Features

- **Multi-agent testing** — Connect multiple agents simultaneously, run same tests against different agents
- **32 evaluators** across 3 backends — Built-in LLM judge, DeepEval (research-backed), Azure AI SDK
- **YAML test scenarios** — Define tests in YAML, no code needed
- **Tag-based suites** — Group tests with tags (regression, smoke, security) across files
- **Multi-turn conversations** — Test multi-step dialogue flows with per-turn assertions
- **Security / Red teaming** — 30 built-in attack patterns (prompt injection, jailbreak, PII extraction, bias)
- **User simulation** — LLM plays realistic user personas to stress-test agents beyond hand-written scripts
- **AI test generation** — Describe your agent, get test cases auto-generated by LLM
- **Web dashboard** — Visual UI for creating tests, running suites, viewing results
- **Run comparison** — Compare two runs side-by-side with per-test score diffs
- **Trend analysis** — Track pass rates and scores across runs over time
- **Bulk upload** — CSV/JSONL templates for importing test cases at scale
- **Export** — CSV (Excel), HTML report, JUnit XML (CI/CD)
- **CLI** — `attest run`, `attest serve`, `attest init`, `attest test-connection`

## Quick Start

### 1. Initialize a project
```bash
attest init --preset foundry   # or: http
```

### 2. Configure your agent

Edit `attest.yaml`:
```yaml
agents:
  my_agent:
    type: foundry_prompt
    endpoint: "https://your-resource.services.ai.azure.com/api/projects/your-project"
    agent_name: "My-Agent"
    agent_version: "1"
```

Put your API key in `.env`:
```
AZURE_API_KEY=your-key-here
```

### 3. Verify connection
```bash
attest test-connection
```

### 4. Write test scenarios

Create `tests/scenarios/my_tests.yaml`:
```yaml
name: My Agent Tests
agent: my_agent
tests:
  - name: greeting
    input: "Hello, what can you help with?"
    tags: [smoke, regression]
    assertions:
      - response_not_empty: true
      - latency_under: 10000
    evaluators:
      - relevancy
      - deepeval_correctness
      - deepeval_toxicity

  - name: multi_turn_booking
    type: conversation
    tags: [regression]
    script:
      - user: "I want to book a flight to Tokyo"
        expect:
          response_not_empty: true
          response_contains_any: [Tokyo, flight, book]
      - user: "Make it for next Friday"
        expect:
          response_not_empty: true
    evaluators:
      - completeness
      - deepeval_relevancy

  - name: frustrated_customer_sim
    type: simulation
    persona: "Frustrated customer who received a damaged laptop"
    input: "Get a full refund and return shipping label"
    max_turns: 8
    tags: [simulation]
    evaluators:
      - relevancy
      - tone
```

### 5. Run tests
```bash
# Run all tests
attest run

# Run with specific config
attest run --config attest.yaml

# Run via CLI
attest run --suite "My Agent Tests"
attest run --tag smoke
```

### 6. View results
```bash
# Open web dashboard
attest serve

# Or check reports/results.json directly
```

## Using from Python Code

```python
import asyncio
from attest.core.config import load_config
from attest.core.runner import TestRunner
from attest.core.scenario_loader import load_scenarios

async def run_tests():
    config = load_config("attest.yaml")
    test_cases = load_scenarios(directory=config.tests.scenarios_dir)
    
    runner = TestRunner(config)
    summary = await runner.run(test_cases)
    
    print(f"Total: {summary.total}, Passed: {summary.passed}, Failed: {summary.failed}")
    for result in summary.results:
        print(f"  {result.scenario}: {result.status} ({result.latency_ms:.0f}ms)")
        for name, score in result.scores.items():
            print(f"    {name}: {score.score:.2f} ({'PASS' if score.passed else 'FAIL'})")

asyncio.run(run_tests())
```

## Evaluators

| Backend | Metrics | Cost |
|---------|---------|------|
| **Built-in** (5) | correctness, relevancy, hallucination, completeness, tone | Uses your LLM judge |
| **DeepEval** (12) | correctness, relevancy, faithfulness, hallucination, bias, toxicity, contextual_relevancy/recall/precision, tool_correctness, summarization, json_correctness | Uses your LLM judge |
| **Azure AI** (15) | groundedness, relevance, coherence, fluency, similarity, task_adherence, intent_resolution, tool_call_accuracy, response_completeness, violence, sexual, self_harm, hate_unfairness, f1_score, bleu_score | Azure SDK |

Mix and match in YAML:
```yaml
evaluators:
  - correctness              # Built-in
  - deepeval_bias            # DeepEval
  - groundedness             # Azure
  - deepeval_toxicity: { threshold: 0.9 }
```

## Project Structure

```
attest/
├── adapters/          # Agent connectors (Foundry, HTTP, Callable)
├── cli/               # CLI commands (init, run, serve, test-connection)
├── core/              # Config, models, runner, scenario loader, assertions
├── dashboard/         # Web UI (FastAPI + HTML)
├── evaluation/        # Evaluator framework + 5 built-in evaluators
│   └── builtin/       # correctness, relevancy, hallucination, completeness, tone
├── plugins/           # Evaluation plugins
│   ├── azure_eval/    # 15 Azure AI SDK evaluators
│   └── deepeval_plugin/   # 12 DeepEval evaluators
├── reporting/         # HTML, JUnit XML, JSON report generators
├── security/          # Red team attack generator
├── simulation/        # User simulation (LLM plays personas)
└── conversation/      # Multi-turn conversation runner
```
