Upload Test Cases

Upload a CSV or JSONL file to bulk-create test cases. Tests are grouped by suite into YAML files.
📥 Download a template, fill it out, then upload:
📄 CSV Template (all test types) 📋 JSONL Template (all test types)
CSV Columns
name, suite, input, expected_output, context, tags, assertions, evaluators, type, persona, max_turns
Test Types (type column)
single_turn — single message → response (default)
conversation — multi-turn script (JSONL only)
simulation — LLM plays user persona vs agent
Simulation Fields
persona — who the simulated user is
max_turns — max conversation turns (default 8)
input — the goal the simulated user tries to achieve
RAG / Context Fields
context — source document text for grounding evaluation
expected_output — ideal answer for correctness scoring
Assertions (semicolon-separated)
response_not_empty · response_contains:text · response_not_contains:text
response_contains_any:a,b,c · response_matches_regex:pattern · exact_match:text
latency_under:5000 · token_usage_under:500
tool_called:name · tool_not_called:name · no_tool_called
response_is_json · json_field_exists:name,email · classification:pos,neg
Evaluators (comma-separated)
Built-in: correctness · relevancy · hallucination · completeness · tone
DeepEval: deepeval_correctness · deepeval_relevancy · deepeval_faithfulness · deepeval_toxicity · deepeval_bias
Azure: groundedness · coherence · fluency · task_adherence · violence
